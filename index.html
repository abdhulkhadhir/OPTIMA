The Problem We're Solving

Traditional motorway control systems rely on manually calibrated algorithms that struggle to adapt to dynamic traffic conditions. This leads to:

  • Inefficient queue detection and management
  • High false positive rates in congestion alerts
  • Manual parameter tuning requiring extensive expertise
  • Limited integration of emerging data sources

Our Innovative Approach

OPTIMA leverages cutting-edge technology and data science to create intelligent, self-optimizing traffic management systems that:

  • Automatically calibrate algorithm parameters
  • Integrate multiple data sources for enhanced accuracy
  • Reduce manual intervention by 80%
  • Improve traffic flow efficiency and safety

Research Methodology

Four comprehensive phases from algorithm documentation to automated optimization

01

Algorithm Documentation & Analysis

Comprehensive review of Queue Detection, Queue Protection, and High Flow algorithms

Queue Detection (QD) Algorithm

Detects queue formation using speed and occupancy thresholds across multiple detector sites. Key parameters include consecutive intervals, consecutive sites, and vehicle length specifications.

Queue Protection (QP) Algorithm

Implements Variable Speed Limit control to protect detected queues. Utilizes queue growth speed calculations and upstream traffic management strategies.

High Flow (HF) Algorithm

Proactively prevents congestion by monitoring flow thresholds and implementing preemptive speed reductions during high-demand periods.

02

Sensitivity Analysis & Simulation

Systematic evaluation of algorithm parameters using advanced traffic simulation

Study Corridor Configuration

The Centenary Highway corridor provides an ideal testbed with 17 detector sites and 9 LUMS installations across 7 kilometers, enabling comprehensive parameter analysis.

Key Sensitivity Analysis Findings

🔴 Highly Sensitive QD-QP Parameters
Consecutive Intervals (CI)

Most critical parameter affecting detection timing and false positive rates

Recommended: 3 Consecutive Intervals (CI) for optimal balance
Consecutive Sites (CS)

Essential for spatial validation of queue presence

Recommended: 2 Consecutive Sites (CS) for low detector spacing
Average Vehicle Length

Directly impacts occupancy threshold calculations

Recommended: Site-specific calibration required
Activation/Deactivation Thresholds

Speed and occupancy limits for queue detection

Recommended: Dynamic threshold adjustment
Average Time Headway

Critical for flow calculations and congestion identification

Recommended: 2.0 seconds for urban motorways
🟡 Moderately Sensitive Parameters
Initial Queue Growth Speed

Affects queue propagation rate calculations

Recommended: Keep current default settings
🔵 HF Algorithm Critical Parameters
Flow Thresholds

Most sensitive HF parameter determining activation timing

Recommended: Site-specific calibration using historical data
Persistence Interval

Time duration for maintaining high flow conditions

Recommended: 20-second intervals for enhanced responsiveness

Critical Recommendations from Sensitivity Analysis

🎯
CS-CI Interaction Optimization

Optimal combination of CS=2, CI=3 for corridors with low detector spacing reduces false positives while maintaining high detection accuracy.

⚡
Use 20-Second Intervals for HF

Observable performance improvement with 20-second intervals instead of 60-second intervals, enhancing responsiveness to traffic changes.

🔧
Localize CS Parameter

Adapt Consecutive Sites parameter to site-specific detector spacing to minimize false positives in dense detection networks.

✅
Retain Default Logic Conditions

Default activation condition (AND) and deactivation condition (OR) are logically sound and perform optimally.

03

Enhanced Traffic State Estimation

Advanced data fusion and machine learning techniques for superior traffic state estimation

Multi-Source Data Integration

CAVI and Compass IoT trajectory data enhance traditional loop detector capabilities, providing comprehensive traffic state information.

Dynamic Time Warping (DTW)

Advanced lane identification using trajectory centerline estimation, enabling precise vehicle positioning and flow analysis.

Adaptive Smoothing Method (ASM)

Spatially interpolates sparse detector data to create continuous speed heatmaps, revealing traffic patterns invisible to traditional methods.

Canny Edge Detection

Computer vision techniques identify queue boundaries from speed heatmaps, enabling precise congestion detection and measurement.
